# 投喂小厨房

投喂小厨房是一款面向双人场景的点餐与接单小应用。应用围绕“一个人点餐、一个人投喂/接单”的使用方式设计，支持房间隔离、点餐下单、主厨接单、菜单管理、偏好提醒、订单历史和首页投喂洞察。

当前项目已经完成第一次归档：现阶段所需功能均已开发完成，暂无新的功能需要继续开发。归档记录见 [ARCHIVE.md](./ARCHIVE.md)。

## 功能总览

### 登录、房间与身份

- 支持微信登录；H5 环境下会使用模拟登录，方便本地调试。
- 支持创建 6 位房间号，每个房间拥有独立菜单、订单、收藏和用户偏好。
- 支持加入历史房间或输入房间号加入指定房间。
- 支持两种身份：
  - 点餐端：浏览菜单、加入投喂清单、提交订单、查看记录。
  - 主厨端：接单、管理订单、维护菜单、查看投喂数据。
- 支持点餐端和主厨端之间切换。

### 点餐端

- 首页展示问候语、每日情诗、房间号和头像入口。
- 首页提供“专属投喂洞察”：
  - 最近最爱：根据近 30 天订单统计 Top 菜品；无近 30 天数据时回退全部订单；无订单时用收藏菜品兜底。
  - 饭点时钟：按深夜、上午、中午、下午、晚上统计常见下单时间。
  - 安心避雷：只展示用户明确设置的“绝对不吃”和“过敏提醒”。
- 首页支持随机抽菜，适合不知道吃什么时快速决定。
- 点餐页支持分类筛选、关键词搜索、菜品数量展示和购物车入口。
- 菜品详情页支持查看图片、描述、选项、收藏和加入投喂清单。
- 投喂清单支持数量调整、删除菜品、备注、提交订单。
- 下单时会根据用户偏好检查风险项，并把提醒带给主厨端。
- 清单总览支持日历热力图、按日查看订单、全部订单时间线和订单详情。
- 订单详情支持查看状态流转、菜品选项、备注和催单。
- 个人页支持头像、投喂次数、收藏数量、特权兑换券、绝对不吃、过敏提醒、清除当前房间数据和退出房间。

### 主厨端

- 接单中心展示当前房间号、今日投喂数据、待接单/制作中/已完成统计。
- 支持查看最新待处理订单，并从首页快速接单。
- 订单管理支持待处理、制作中、已完成等状态视图。
- 支持订单状态流转：
  - 待接单
  - 已接单
  - 制作中
  - 已完成
- 主厨端可查看用户备注、菜品选项、风险提醒和催单提醒。
- 菜品初始化页支持首次创建默认菜单。
- 菜品管理页支持新增、编辑、删除菜品，上传图片，管理分类，上下架菜品。
- 主厨个人页支持头像、订单入口、菜单入口、切换点餐端和退出房间。

### 偏好、避雷与特权

- 用户可维护“绝对不吃”和“过敏提醒”。
- 下单时会对购物车菜品的名称、描述、选项、标签等进行匹配，生成风险提醒。
- 风险提醒会随订单展示给主厨，减少误投喂。
- 支持自定义特权兑换券。
- 特权券可按投喂次数解锁、兑换和展示状态。

### 数据同步与云开发

- 微信小程序环境优先使用微信云开发。
- H5 环境使用本地模拟数据和本地缓存，便于开发调试。
- 菜单、分类、订单、房间、用户偏好和特权券按房间隔离。
- 菜单支持实时监听；不可用时使用轮询兜底。
- 订单状态更新支持云端同步；本地会做乐观更新和状态优先级保护，避免状态回退。
- 当前包含的云函数目录：
  - `login`
  - `joinRoom`
  - `getUserRooms`
  - `getMenuItems`
  - `updateMenuItem`
  - `updateOrder`
  - `updateRoomPreferences`

## 本地运行

安装依赖：

```bash
npm install
```

运行 H5：

```bash
npm run dev:h5
```

构建 H5：

```bash
npm run build:h5
```

运行微信小程序：

```bash
npm run dev:mp-weixin
```

构建微信小程序：

```bash
npm run build:mp-weixin
```

## 小程序云开发说明

微信小程序端需要配置并部署云开发环境。云环境 ID 配置在 `src/services/cloud.js` 中。

首次在微信小程序端运行前，建议确认：

- 云开发环境已开通。
- 必要云函数已部署。
- 数据库集合权限符合项目读写需求。
- 菜品图片上传目录可用。
- 房间、菜单、订单、用户偏好等集合能正常创建或访问。

H5 端不会真实访问微信云开发，会使用本地模拟登录、本地默认菜单和本地订单缓存。
